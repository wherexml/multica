# Lessons

- No session-specific lessons recorded yet.
- Docker redeploys must use `.env.container`; the default `.env` is for local dev and can silently put the container stack on the wrong ports and auth origins.
- If the runtime image keeps `ENTRYPOINT ["./server"]`, the compose `migrate` service must override entrypoint to `./migrate`; otherwise `docker compose run migrate` starts the server and hangs instead of applying migrations.
- The active deployment port range for this workspace must match the current task target exactly; `2200X` and `2220X` both exist in repo history, so always verify the requested frontend URL before rebuilding containers or repointing the daemon.
- Before changing daemon or local URLs, inspect the running backend container labels first. This workspace has carried multiple local port conventions in repo history, so the fix is to verify the active compose/env source before repointing anything.
- After rebuilding the local daemon, always check its health endpoint agent list and `/api/runtimes`; the source can silently miss a provider (for example `Gemini`) even when the CLI binary is installed, which turns a port-only fix into a runtime regression.
- **Sidebar invisible fix**: If the left sidebar is completely missing (even in incognito), check `apps/web/app/(dashboard)/layout.tsx`. It must include `AppSidebar` inside `SidebarProvider`. The upstream or some local edits may have removed it, leaving only `SidebarInset` (main content). Always ensure `AppSidebar` is rendered alongside `SidebarInset` in the dashboard root layout.
- On private HTTP origins such as `http://10.x.x.x`, browsers may expose `crypto.getRandomValues` but not `crypto.randomUUID`; API code must not require `randomUUID` before sending login requests, or the UI can show a misleading auth error without the backend receiving `/auth/login`.
- Runtime connection scripts must support a manual callback URL fallback because released CLI binaries may generate callback hosts that are not reachable from the user's browser in VM or remote-shell setups; `--callback-url` lets the script finish login from the final browser URL.
- Orb/non-interactive shell commands may not load `.bashrc`/NVM paths, so an agent CLI can be installed but invisible to `multica daemon start`; runtime connection scripts should prepend common user CLI directories such as `~/.npm-global/bin`, `~/.local/bin`, and NVM `bin` paths before starting the daemon.
